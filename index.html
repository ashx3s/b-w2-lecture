Plan then Code Methodology

Steps to use when working though most coding projects

  1. Project Setup

    • Identify what your goal is (what do you want to make)
    • Identify constraints and limitations (scope). where should you stop?
    • Identify unknowns. topics that might be relevant but you aren't confident with

  2. Plan it out

    • Use comments to map out the blocks of code
    • If we need content, use AI / our creativity to make stuff up... or research depending the nature of the project
    • Sketch out what the page should look like (very low fidelity)
    • Look for inspiration

  3. Write your code!

    • Go top to bottom and write out the content with appropriate tags
    • make commits regularly
    • Check that the code is working
    • If it's not working, read and think through why. critique your code ruthlessly
    • After finishing one of tasks, step back and evaluate

Important Syntax

  • tr - table row
  • th - table heading
  • td - table data

Other Syntax to review (with current working assumptions)

  • caption - a label for the table
  • thead - a table head container?
  • tbody - a container for main table content
  • tfoot - the footer

Common Mistakes

  • Using p2 or other numbered p tags: there is only 1 level of p tag
  • using b or i tag for bold and italic
  • This is wrong, no p2, use just p
  • Don't use capitals in tag names
  • Using li tag without wrapping it in an ol or ul
  • not setting up the code with an ! hotkey (no metadata etc)
  • naming the index.html file anything except for index.html
  • Putting visible content outside of the body tags
