Plan then Code Methodology

Steps to use when working though most coding projects

Project Setup

  1. Identify what your goal is (what do you want to make)
  2. Identify constraints and limitations (scope). where should you stop?
  3. Identify unknowns. topics that might be relevant but you aren't confident with

Plan it out

  1. Use comments to map out the blocks of code
  2. If we need content, use AI / our creativity to make stuff up... or research depending the nature of the project
  3. Sketch out what the page should look like (very low fidelity)
  4. Look for inspiration

Write your code!

  1. Go top to bottom and write out the content with appropriate tags
  2. make commits regularly
  3. Check that the code is working
  4. If it's not working, read and think through why. critique your code ruthlessly
  5. After finishing one of tasks, step back and evaluate

Common Mistakes

  • Using p2 or other numbered p tags: there is only 1 level of p tag
  • using b or i tag for bold and italic
  • This is wrong, no p2, use just p
  • Don't use capitals in tag names
  • Using li tag without wrapping it in an ol or ul
  • not setting up the code with an ! hotkey (no metadata etc)
  • naming the index.html file anything except for index.html
  • Putting visible content outside of the body tags
